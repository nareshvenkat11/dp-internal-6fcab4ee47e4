Architecture
SPEC

Two-Orchestrator Architecture

Fabric AI Orchestrator (user-facing) delegates to DataProjects Orchestrator (backend) via HTTP REST API. CLI and In-Fabric share one engine.

SPEC

CLI Experience

DataProjects agent in GitHub Copilot, Claude Code, and Cursor. Discovery via /api/v1/hello, project lifecycle through REST API.

SPEC

In-Fabric Experience

Project Mode toggle in the Fabric AI Orchestrator. DataProjects Panel for milestones, production monitors, and review.

API

API Interface Specification

Full REST API between orchestrators. Discovery, project CRUD, milestones, status protocol, webhooks.

Building Blocks (Concepts)
CONCEPT

Projects

Workspace-scoped logical constructs with persistent state, team collaboration, and governance. One shared AI Projects Lakehouse per workspace.

CONCEPT

Milestones

The core delivery unit. Assess > Plan > Build > Validate > Deploy. Per-milestone review, approval, and production deployment.

CONCEPT

Production Monitors

Auto-deployed with each approved milestone. Row-count trending, schema drift detection, null-rate anomalies, freshness SLA tracking.

CONCEPT

My Projects vs Team Projects

Binary toggle. Private by default (My Projects). Workspace RBAC when shared (Team Projects). No separate permission model.
